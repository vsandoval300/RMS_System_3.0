Alta Manual de Negocio

📍 RMS Platform 4 · Underwritten Module Camino 1 Alta individual por interfaz gráfica 6 pasos · flujo secuencial
1

Crear el Negocio (Business)

Registro principal del contrato de reaseguro. Es la raíz de toda la jerarquía.
Businesses › + New Business
Identificación
business_code description index source_code
Contraparte & Alcance
reinsurer producer currency region
Atributos del Contrato
reinsurance_type risk_covered business_type premium_type purpose claims_type
Vínculos opcionales
parent (treaty) renewed_from
ℹ️ premium_type define el comportamiento del ciclo financiero: Fixed = monto fijo; Estimated / Declared = requiere recalculaciones periódicas por bordereaux.
Resultado Registro businesses creado · approval_status = DFT · lifecycle = On Hold
2

Crear Cost Scheme (Estructura de Costos)

Define cómo fluye la prima entre las partes y qué deducciones aplican. Puede reutilizarse en varios negocios.
Cost Schemes › + New
Encabezado del Scheme
scheme_id agreement_type share (%) index description
Cost Nodes (Repeater — dentro del mismo form)
deduction concept partner source partner destination value apply to gross
Agrega un nodo por cada tramo del flujo premium → comisión → neto. El total de value de todos los nodos debe sumar 100%.
⚠️ El Cost Scheme es independiente del Business. Se vincula al Operative Doc en el Paso 4 (tab "Doc Schemes") y a cada Insured también en el Paso 4.
Resultado Registro cost_schemes + N registros cost_nodesx
3

Agregar Liability Structures (Coberturas)

Define los límites de cobertura del negocio. Una fila por tipo de cobertura.
Business › Edit › Tab "Liability Structures" › + Add
⬆ Requiere: Business creado (Paso 1)
Por cada cobertura
coverage limit limit description CLS (Yes/No) sublimit sublimit desc deductible deductible desc
ℹ️ CLS = Combined Limit with Sublimit. Activarlo indica que el sublímite está contenido dentro del límite principal, no es adicional.
Resultado N registros liability_structures vinculados al Business
4

Agregar Operative Document

El documento operativo es la póliza concreta del negocio. El modal tiene 4 pestañas que deben llenarse en orden.
Business › Edit › Tab "Operative Documents" › + Add
⬆ Requiere: Business (Paso 1) + Cost Scheme (Paso 2)
1
Document Details Datos generales del documento
document ID description document type AF/MF ROE/FS rep date
2
Coverage Period Vigencia de la cobertura
inception date expiration date
El sistema calcula automáticamente la duración en días.
3
Insured Members Asegurados y sus primas
Repeater — un renglón por asegurado:
company (asegurado) coverage cost scheme premium
ℹ️ El cost scheme seleccionado aquí por cada insured es el que usará TransactionLogsPreviewService para calcular los logs financieros al crear una transacción.
4
Doc Schemes Asignación del scheme al documento
Vincula el Cost Scheme a este Operative Doc (tabla businessdoc_schemes):
cost scheme
El scheme aquí define la estructura de distribución a nivel documento. Normalmente coincide con el del insured.
Resultado operative_docs + businessdoc_insureds + businessdoc_schemes · lifecycle → In Force
5

Agregar Transacciones

Cada installment financiero del documento. Al guardarse, genera automáticamente los TransactionLogs por partida.
Operative Doc › View › + New Transaction
⬆ Requiere: Operative Doc completo (Paso 4)
Por cada installment
transaction type due date amount proportion (%) exchange rate remittance code
⚡
Al guardar la transacción, TransactionLogsPreviewService se dispara automáticamente y genera los Transaction Logs (una línea por partida por contraparte según el Cost Scheme de cada insured). ⚙ auto-generado
Resultado transactions + transaction_logs · status inicial = Pending
6

Someter a Revisión y Aprobar

Cuando el negocio esté completo, el Creator lo envía al Manager para su aprobación. Solo los negocios APR aparecen en dashboards y reportes.
Business › Edit / View › "Submit for Review"
⬆ Solo disponible cuando status = DFT o REJ
1️⃣
Creator presiona "Submit for Review"
status DFT → PND · notificación campanita al Manager (inmediata)
2️⃣
Manager recibe resumen semanal por correo (lunes 8am) o revisa la campanita
Entra a Business → View/Edit → acción en header
3️⃣
Opción A — Approve: status PND → APR · correo inmediato al Creator
El Business aparece en dashboards, reportes y PremiumForPeriodService
4️⃣
Opción B — Request Revision: status PND → REJ · correo con revision_notes al Creator
Creator corrige y vuelve a "Submit for Review" (loop hasta Approve)
Terminal approval_status = APR · visible en Underwritten Dashboard y todos los reportes
